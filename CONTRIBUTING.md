# 贡献指南

感谢你愿意分享经验!正是一份份真实的分享,让这个项目对后来人有价值。请花两分钟读完本指南。

## 一、贡献流程

### 方式 A:提交 Pull Request(推荐,熟悉 Git 的同学)

1. Fork 本仓库
2. 从 [`templates/`](templates/) 复制对应模板到正确的目录
3. 按模板填写内容,文件命名遵循下方规范
4. 提交 PR,维护者 review 后合并

### 方式 B:提 Issue(不会 Git 也能参与)

直接新建一个 Issue,把你的内容按模板贴进去,注明应该放到哪个学院/主题下,维护者会帮你整理入库。

## 二、内容该放哪里

| 你的分享 | 放置位置 |
|----------|----------|
| 就业经验(求职、实习、offer) | `学院/<学院名>/就业/经验/` |
| 保研经验 | `学院/<学院名>/保研/经验/` |
| 考研经验 | `学院/<学院名>/考研/经验/` |
| 考试资料 / 课程笔记 | `学院/<学院名>/考试资料/` |
| 转专业经验 | `转专业/经验/` |
| 学校生活类(选课、住宿、办事等) | `学校生活/` 下对应子目录 |

> 各目录下的 `README.md` 维护的是**客观汇总信息**(如统一的时间节点、统计数据);
> `经验/` 文件夹里放的是**个人主观分享**。两者分工明确,不要把个人经验写进 README。

## 三、文件命名规范

经验类文件统一格式:

```
年份-昵称-方向-去向.md
```

示例:

- `2025-小明-后端开发-某互联网公司.md`
- `2025-Alex-保研本校-基础数学.md`
- `2024-阿芳-考研上岸-某985.md`

**年份开头**便于排序和判断时效;**昵称**用 GitHub ID 或自取的代称,**不要用真实姓名**。

## 四、隐私红线 ⚠️(务必遵守)

- ❌ 不写真实姓名、学号、身份证、电话等可定位个人的信息
- ❌ 不发布**他人**的隐私信息(包括同学的成绩、去向、薪资)
- ⚠️ 薪资可写**区间/档位**(如「白菜价」「SP」「20~25w」),避免精确到能反推个人
- ✅ 用昵称 / GitHub ID 代替真名

## 五、版权红线 ⚠️

- ✅ 可以分享:**自己整理的笔记、复习提纲、经验总结、知识点梳理**
- ❌ 不要上传:受版权保护的**教材 PDF、付费课程、机构讲义、完整真题原件**
- 如果引用了他人内容,请注明来源

> 违反版权可能导致整个仓库被下架(DMCA),请自觉遵守。

## 六、写作小建议

- **写时效信息时标注年份**(政策年年变)
- **多写「坑」和「教训」**——这往往比成功经验更有价值
- 能给数据就给数据(报录比、名额数、时间点),比「挺难的」有用得多
- 保持客观,避免对具体老师/同学的人身攻击
